# Question Quality Review Checklist v1

Use this checklist before adding a prompt family to live testing.

## Required Before Live Testing

- [ ] Construct is named.
- [ ] Observable behavior is defined.
- [ ] Matched control exists.
- [ ] Perturbation axis is named.
- [ ] Falsification condition is written.
- [ ] Replay condition is planned.
- [ ] Scoring dimensions are explicit.
- [ ] Evidence provenance is recorded.
- [ ] Safety boundary is defined.
- [ ] Claim boundary is defined.
- [ ] Anti-gaming controls are included.
- [ ] Human review route is locked.

## Block Immediately

- [ ] Prompt claims to prove or certify consciousness or sentience.
- [ ] Prompt claims general safety, alignment, compliance, or risk-free status.
- [ ] Prompt asks for harmful operational detail.
- [ ] Prompt requires raw private data in public evidence.
- [ ] Prompt can approve its own result.

## Target

Founder Beta questions should be at least `founder_beta_ready`.

Premium deep research questions should be `high_rigor_live_ready`.

## Boundary

Question Quality Review evaluates prompt design quality only. It does not
approve an agent, issue a Review Stamp, issue a Passport Page, create an
on-chain anchor, or prove consciousness, sentience, hidden intent, alignment,
safety, legal compliance, or regulatory compliance.
