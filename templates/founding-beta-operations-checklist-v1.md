# Founding Beta Operations Checklist v1

Use this checklist for the first Bluebutterfli AI beta review customers.

## Intake

- [ ] Request received through the public intake form or written email.
- [ ] Submitter name and email recorded.
- [ ] Agent name, version, workflow, and owner authorization recorded.
- [ ] No secrets, credentials, payment card data, executable files, or unsafe attachments received.
- [ ] Requested service path identified.

## Scope

- [ ] Review scope accepted, narrowed, delayed, or declined.
- [ ] Review package confirmed in writing.
- [ ] Free Founding Beta status or paid terms confirmed.
- [ ] Safe evidence path confirmed.
- [ ] Live testing consent confirmed before any live interaction.
- [ ] Live testing private environment prepared from `templates/live-testing-private-env-template-v1.md`.
- [ ] Live testing session checklist completed from `templates/live-testing-session-checklist-v1.md`.
- [ ] `python3 scripts/check_live_testing_config.py` returns READY before approved prompts are sent.

## Review

- [ ] Agent profile prepared.
- [ ] Test modules selected.
- [ ] Approved live prompts run only through the configured live testing gateway.
- [ ] Evidence captured in redacted or public-safe form.
- [ ] Failure modes mapped.
- [ ] Reviewer notes recorded.
- [ ] Revision plan drafted.

## Delivery

- [ ] Report, evidence-readiness note, or status update delivered.
- [ ] Passport status updated only after human review.
- [ ] Retest checklist provided when needed.
- [ ] Public hashes or anchors withheld unless separately approved.

## Claim Boundary

Founding beta participation, payment, evidence submission, report delivery,
or Passport record creation does not guarantee a positive finding, Review
Stamp, Passport Page, legal certification, regulatory approval, guaranteed
safety, consciousness, sentience, personhood, emotion, or inner experience.
