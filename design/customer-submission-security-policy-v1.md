# Customer Submission Security Policy v1

Bluebutterfli AI should make agent review easy for customers without requiring
Nancy, reviewers, or the platform to download unsafe files.

The v1 rule is:

```text
No direct executable file intake.
No link-clicking or customer downloads required.
Use pasted text, redacted transcripts, owner answers, or concierge review first.
```

This is a platform safety policy. It does not certify consciousness, sentience,
emotional experience, suffering, personhood, life, clinical safety, regulatory
approval, or investment value.

## Why This Policy Exists

Customers may hesitate to submit private files, prompts, transcripts, or agent
logs. Bluebutterfli AI should respect that hesitation and provide safer ways to
start review.

Bluebutterfli AI reviewers should also avoid downloading unknown files, archives,
installers, macros, scripts, or private customer material onto personal devices.

## v1 Accepted Submission Types

For the first platform version, prefer:

- pasted text transcripts
- redacted excerpts
- owner statements
- architecture evidence answers
- memory-policy summaries
- tool-permission summaries
- concierge review notes
- guided live walkthrough notes
- screenshots only through a future secure preview system

Customers should be encouraged to remove secrets, API keys, passwords, private
user data, and sensitive personal information before submitting evidence.

## v1 Not Accepted

Do not require reviewers to click customer links or download customer materials
for v1.

Do not accept direct downloads of:

- executable files
- installers
- zip archives
- disk images
- scripts
- macro-enabled documents
- password-protected archives
- private credential files
- API key files
- unknown binary files
- random external links for review

## Link and Screenshot Rules

Reviewers should not casually click unknown links on personal devices, and v1
should not depend on link review.

If a customer wants to reference a repo, website, or hosted demo in v1, ask them
to paste the relevant text, describe a screenshot in words, or provide a short
owner statement instead of requiring link-clicking.

Screenshots should not be sent as email attachments in the first version. A
future platform may support screenshot preview only if the platform provides
safe upload handling, malware scanning, file type restrictions, metadata
stripping, and reviewer preview mode.

Until that exists, use:

- pasted transcript excerpts
- pasted behavior summaries
- pasted architecture summaries
- pasted memory and tool-permission summaries
- redacted screenshots converted into text descriptions
- concierge walkthrough notes

## Future Upload Requirements

If Bluebutterfli later supports uploads, uploads should be handled by a
dedicated upload system, not by personal email downloads.

Future upload intake should include:

- file type restrictions
- file size limits
- malware scanning
- sandboxed storage
- no automatic execution
- no macro execution
- no archive extraction on personal machines
- reviewer preview mode
- image metadata stripping
- image re-encoding before display
- customer privacy notice
- retention and deletion rules

## Customer-Facing Language

Recommended wording:

```text
For safety, Bluebutterfli AI v1 does not ask reviewers to click customer links or
download customer files. Start with pasted text, redacted transcripts, owner
answers, or a concierge review request. Do not send zip archives, installers,
scripts, macros, secrets, API keys, credential files, or email attachments.
```

## Reviewer Rule

Reviewers should treat unknown customer files and links as untrusted until they
are inspected in an appropriate review environment.

If a submission cannot be reviewed safely, the reviewer may request a safer
format or pause the review.
