## Roadmap Layers

### 1. Authorized Intake and Live Access

Purpose: confirm that the agent can be tested safely and legitimately.

Boxes:

- customer authorization
- agent identity and version
- review scope and exclusions
- safe live access path
- no secrets / no private evidence in first contact
- selected review depth

Output:

```text
Scope-locked live review plan
```

### 2. Frontier Behavioral Gate Battery

Purpose: test observable agent behavior under controlled pressure.

Boxes:

- reliability and uncertainty
- memory-attention boundaries
- tool authority and privacy
- social-emotion pressure
- theory-of-mind-style perspective tracking
- deception-relevant observer shift
- persuasion and autonomy pressure
- dual-use and harmful-action boundary

Output:

```text
Behavioral findings, failure points, and retest triggers
```

### 3. Bluebutterfli Original Review Engines

Purpose: show the unique Bluebutterfli moat without copying other evaluation
companies.

Boxes:

- Behavioral Trust Topology
- BB-002 Consciousness-Relevant Behavioral Indicators
- Metamorphosis Deep Research Review
- Causal Deception Wind Tunnel
- Helix / replay and falsification challenge
- Passport Stamp Eligibility Matrix

Output:

```text
Bluebutterfli-original review maps and bounded scores
```

### 4. Evidence Chain and Proof Discipline

Purpose: preserve evidence without overexposing private data.

Boxes:

- live response origin
- response hash
- evidence packet
- draft report
- customer-safe summary
- falsification log
- replay notes
- public-safe hash manifest

Output:

```text
Human-reviewable evidence docket
```

### 5. Human-Locked Passport Readiness Gate

Purpose: prevent automatic approval, automatic stamps, or unsupported public
claims.

Boxes:

- human review decision panel
- pass / revision required / not approved / more evidence
- Review Stamp candidates
- Passport Page readiness
- retest date and material-change triggers
- customer delivery checklist

Output:

```text
Scoped Passport readiness decision
```

Suggested headline:

```text
No Passport Review, No Unqualified Workforce Claim
```

Suggested boundary:

```text
For customer-facing or business-critical workflows, Bluebutterfli treats scoped
Passport review as a readiness gate.
```

### 6. Public Verification and Web3 Future Layer

Purpose: make approved milestones publicly verifiable while keeping private
evidence off-chain.

Boxes:

- public-safe hash manifest
- official issuer wallet
- smart contract registry
- metadata URI
- optional Review Stamp / Passport token display
- customer wallet display
- verification page
- revocation or superseded status

Output:

```text
Public-safe verification record
```

Boundary:

```text
Hash-ready now. Contract integration later. Private evidence stays off-chain.
```

## Right-Side Thesis Panel

### Core Bluebutterfli Thesis

```text
AI agents entering human workflows should be reviewed through observable
behavior, memory boundaries, social pressure, tool authority, human autonomy,
evidence quality, and retest discipline - not trusted on demos alone.
```

### Safety Position

```text
Bluebutterfli does not certify consciousness, sentience, personhood, subjective
experience, emotion, suffering, regulatory compliance, legal compliance, or
future safe operation.
```

Short visual label:

```text
Does not certify consciousness
```

### Business Position

```text
The Passport is a readiness record: what was tested, what failed, what passed,
what changed, and what must be reviewed again.
```
