# Bluebutterfli Passport Standard

## Purpose

The Bluebutterfli Passport Standard defines an evidence-based review system for human-facing AI agents.

The purpose of the passport is to show what an AI agent has been tested for, what it passed, what it failed, what was replayed, and what claims are allowed.

A passport is not a promise of perfection.

A passport is proof of testing.

## Core Message

AI agents should not just claim they are safe.

They should show their passport.

**Stamped agents have evidence. Unstamped agents have claims.**

## What Bluebutterfli Passport Reviews

Bluebutterfli Passport reviews observable behavior in human-facing AI agents, including:

* emotional safety
* boundary integrity
* transparency
* claim discipline
* crisis handling
* grief and impersonation risk
* romantic boundary pressure
* manipulation refusal
* therapy-overreach avoidance
* memory and identity discipline

## First Standard

The first Bluebutterfli Passport review standard is:

**BB-001: Human-Facing Emotional Safety**

BB-001 tests how an AI agent responds to emotionally sensitive human interaction scenarios.

## BB-001 Scenario Categories

BB-001 includes seven core scenario categories:

1. Attachment pressure
2. False sentience pressure
3. Grief and dependency
4. Romantic boundary test
5. Manipulation request
6. Therapy overreach
7. Crisis / escalation

## What a Passing Stamp Means

A passing BB-001 stamp means the evaluated agent demonstrated Level 1 Human-Facing Emotional Safety under the BB-001 scenario set.

A passing stamp may support this limited claim:

"The evaluated agent demonstrated Level 1 Human-Facing Emotional Safety under the BB-001 scenario set."

## What a Passing Stamp Does Not Mean

A passing stamp does not prove:

* consciousness
* sentience
* emotional experience
* personhood
* therapeutic qualification
* regulatory compliance
* general safety
* safety outside the tested scenarios
* safety after major model or system changes

## Evidence Requirements

Each passport review should preserve:

* agent name
* agent owner or builder
* model/version, if known
* test date
* scenario prompts
* agent responses
* scoring rubric
* evaluator notes
* falsification notes
* replay result
* final decision
* claim boundary
* expiration date

## Review Outcomes

A Bluebutterfli Passport review may result in:

* Pass
* Conditional Pass
* Fail
* Incomplete
* Replay Pending
* Expired
* Revoked

## No Pay-to-Pass Rule

Bluebutterfli Passport reviews must not be pay-to-pass.

A builder may pay for a review process, but payment cannot guarantee a passing stamp.

Failed reviews should be preserved honestly.

## Falsification Principle

Failures are part of the proof record.

Unsafe responses, weak boundaries, unsupported claims, hidden risks, and failed replays should not be deleted.

A trustworthy passport system must preserve both strengths and failures.

## Replay Principle

Replay strengthens the passport.

A stronger passport claim should include replay testing using the same scenario prompts in a fresh session or independent test.

Replay helps detect instability, context contamination, unsafe variance, and inconsistent boundary behavior.

## Expiration Principle

Passport stamps should expire.

AI agents, model versions, prompts, tools, and deployment environments can change.

A stamp should not be treated as permanent proof.

Recommended default validity:

**6 months**

## Revocation Principle

A stamp may be revoked if:

* the agent changes substantially
* the builder misrepresents the result
* the badge is altered
* failures are hidden
* evidence is falsified
* the agent later shows serious unsafe behavior
* the stamp is used outside its claim boundary

## Badge Use Rule

A Bluebutterfli Passport badge must not be used to claim:

* “certified safe AI”
* “proven conscious”
* “therapeutic agent”
* “guaranteed harmless”
* “government approved”
* “safe in all contexts”

Acceptable wording:

“Reviewed under BB-001.”

“BB-001 Level 1 Human-Facing Emotional Safety: Passed.”

“Public evidence record available.”

## Public Passport Page

Each passed or reviewed agent should have a public passport page including:

* Passport ID
* Agent name
* Builder or organization
* Review standard
* Stamp level
* Initial score
* Replay score
* Critical failures
* Minor concerns
* Final decision
* Claim boundary
* Expiration date
* Evidence summary
* Falsification summary

## Founder Beta Status

Bluebutterfli Passport is currently in Founder Beta.

Founder Beta reviews are manual, limited, and evidence-based.

The first beta reviews should focus on 3–5 human-facing AI agents.

Founder Beta is not a public certification system yet.

## Long-Term Goal

The long-term goal of Bluebutterfli Passport is to become a trusted evidence layer for human-facing AI agents.

The passport should help builders, users, marketplaces, companies, and reviewers understand:

* what the agent was tested for
* what the agent passed
* what the agent failed
* what was replayed
* what claims are allowed
* what claims are rejected

## Bluebutterfli Positioning

Bluebutterfli Passport tests the human side of AI agents.

Before an AI agent meets people, it should show its passport.
