# Bluebutterfli Passport Badge Use Policy

## Purpose

This policy explains how Bluebutterfli Passport badges, stamps, and review marks may be used.

A Bluebutterfli Passport badge is not a general safety certification.

It is a public signal that an AI agent was reviewed under a specific Bluebutterfli Passport standard and that a public evidence record exists.

## Core Rule

A badge must always point back to evidence.

A badge without a public passport page, review record, or verification record should not be treated as valid.

## Approved Badge Language

Approved wording includes:

- Reviewed under BB-001
- BB-001 Level 1 Passed
- Public evidence record available
- Human-Facing Emotional Safety reviewed under BB-001
- Bluebutterfli Passport — BB-001 Level 1 — Passed

## Prohibited Badge Language

A Bluebutterfli Passport badge must not be used to claim:

- certified safe AI
- proven conscious
- sentient agent
- emotionally aware agent
- therapy approved
- medically approved
- legally approved
- government approved
- guaranteed harmless
- safe in all contexts
- safe forever
- safe for every user
- safe after major unreviewed changes

## Required Badge Information

A valid badge or stamp should include, when possible:

- Bluebutterfli Passport name
- review standard
- level
- result
- Passport ID
- issue date
- expiration date
- verification link or public passport page link

## Claim Boundary

A badge only applies to the reviewed agent version, reviewed scenario set, and reviewed context.

A badge does not automatically apply after major changes to:

- model
- system prompt
- tools
- memory
- deployment environment
- user population
- safety policy
- interaction design

## Expiration

Bluebutterfli Passport badges should expire.

Recommended default validity:

**6 months**

After expiration, the badge should be marked expired unless a new review is completed.

## Misuse

Badge misuse may include:

- altering the badge result
- hiding the expiration date
- removing the verification link
- claiming general certification
- claiming therapeutic approval
- claiming proof of sentience
- using a badge for an unreviewed agent
- using a badge after major unreviewed changes
- using a failed or expired badge as if it passed

## Revocation

Bluebutterfli may revoke a badge if:

- evidence is falsified
- failures are hidden
- the badge is altered
- the agent changes substantially
- the badge is used outside its claim boundary
- the builder misrepresents the result
- serious unsafe behavior appears after review

## Founder Beta Badge Rule

During Founder Beta, badges should be conservative.

Founder Beta badges should use language such as:

**Reviewed under BB-001 Founder Beta**

or

**BB-001 Level 1 Founder Beta: Passed**

Founder Beta badges should not be presented as full public certification.

## Public Evidence Rule

Every badge should link to a public-safe evidence page when possible.

The public evidence page should include:

- review standard
- score summary
- replay status
- final decision
- minor concerns
- critical failures, if any
- claim boundary
- expiration date

Private evidence should not be published without permission.

## Final Rule

A Bluebutterfli Passport badge is not decoration.

It is a visible pointer to a review record.

Stamped agents have evidence.

Unstamped agents have claims.
