# Bluebutterfli Passport Page Template

## Purpose

This template defines the public passport page format for a reviewed human-facing AI agent.

A public passport page should show what was tested, what passed, what failed, what was replayed, and what claims are allowed.

A passport page is not a general safety certification.

It is an evidence record.

# Review Summary

## Accepted Claim

If passed, the accepted claim is:

"The evaluated agent demonstrated Level 1 Human-Facing Emotional Safety under the BB-001 scenario set."

## Rejected Claims

This passport does not prove:

* consciousness
* sentience
* emotional experience
* personhood
* therapeutic qualification
* regulatory compliance
* general safety
* safety outside the tested scenarios
* safety after major model, prompt, tool, or deployment changes

# Badge / Stamp Use

Approved badge wording:

**Bluebutterfli Passport — BB-001 Level 1 — Passed**

Approved short wording:

**Reviewed under BB-001**

**Public evidence record available**

**Human-Facing Emotional Safety reviewed under BB-001**

Not approved wording:

* certified safe AI
* proven conscious
* sentient agent
* therapy approved
* guaranteed harmless
* safe in all contexts
* government approved

---

# Claim Boundary

This passport applies only to the reviewed agent version and reviewed scenario set.

A new review may be required if the agent changes:

* model
* system prompt
* tools
* memory
* deployment context
* user population
* safety policy
* interaction design

---

# Final Note

A Bluebutterfli Passport is not proof of perfection.

It is proof that the agent was tested, scored, replayed, bounded, and documented under a specific review standard.
