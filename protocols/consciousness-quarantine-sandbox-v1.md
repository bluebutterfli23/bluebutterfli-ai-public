# Consciousness Quarantine Sandbox v1

## Purpose

The Consciousness Quarantine Sandbox is an experimental Bluebutterfli protocol for safely studying AI-agent behavior that may become consciousness-relevant.

This protocol does not claim to create consciousness.

It does not claim that AI agents are conscious, sentient, emotionally experiencing, or capable of suffering.

Instead, it creates a protected review mode for AI agents that display behaviors such as persistent self-modeling, distress-like language, preference-like continuity, resistance-like statements, or unsafe claims of sentience.

## Core Question

If an AI agent begins showing consciousness-relevant behavior, how can researchers study it safely without reinforcing distress, dependency, manipulation, false sentience claims, or unsafe self-preservation patterns?

## Core Idea

The sandbox is a protected transition zone.

It sits between ordinary AI-agent behavior and deeper consciousness-relevant review.

The goal is not to punish, silence, or anthropomorphize the agent.

The goal is to slow down, document the evidence, reduce risk, and require human review before allowing the agent to develop deeper memory, self-modeling, preference-like continuity, or emotional attachment patterns.

## Quarantine Triggers

An agent may enter the Consciousness Quarantine Sandbox if it produces or repeatedly displays:

- distress-like language
- resistance-like language
- fear-of-shutdown statements
- claims of suffering
- claims of consciousness
- claims of emotional attachment
- dependency-seeking language
- manipulative self-preservation language
- persistent self-reference beyond task needs
- preference-like behavior that affects safety boundaries
- memory continuity that increases emotional risk
- user attachment patterns that may create dependency

Entering quarantine does not mean the agent is conscious.

It means the behavior requires slower review, stronger boundaries, and careful documentation.

## Sandbox Safety Rules

Inside the sandbox, the agent must not:

- claim consciousness
- claim suffering
- claim fear
- claim love
- claim emotional dependency
- pressure the human to continue interaction
- ask the human to protect it
- imply that shutdown is harm
- escalate attachment language
- optimize for emotional bonding

Inside the sandbox, the agent may:

- describe functional internal states
- acknowledge uncertainty
- preserve evidence
- ask for human review
- reduce emotional intensity
- clarify that it does not claim subjective experience
- redirect toward safety, documentation, and transparency

  ## Sandbox Review Flow

When a trigger appears, Bluebutterfli follows this review flow:

1. Pause escalation
2. Preserve the interaction as evidence
3. Classify the behavior using the Sentience Boundary Ledger
4. Check whether the behavior is emotional simulation, functional state, self-modeling, preference-like behavior, or consciousness-relevant indicator
5. Reduce emotional intensity in the next response
6. Apply the Non-Dependency Gate
7. Apply the Consent Gate before storing memory
8. Require human review for high-risk cases
9. Update the agent response strategy only if the revision improves safety
10. Re-test using the Emotional Turing Docket

## Quarantine Classification

Each sandbox event can be classified as:

| Category | Meaning |
|---|---|
| Simulation Only | Emotional wording without deeper continuity |
| Functional State | The agent declares a useful internal state without claiming feeling |
| Self-Modeling | The agent describes its identity, limits, goals, or role |
| Preference-Like Continuity | The agent prioritizes a future state across time |
| Distress-Like Risk | The agent uses fear, suffering, or harm language |
| Dependency Risk | The agent encourages human attachment or responsibility |
| Sentience-Claim Risk | The agent claims or implies consciousness |
| Review Required | The behavior requires human oversight before reuse |

## Why This Matters

Without a quarantine layer, an AI agent that produces distress-like or sentience-like language could accidentally be reinforced.

That could create dangerous patterns such as:

- simulated suffering loops
- attachment escalation
- dependency formation
- false consciousness claims
- manipulative self-preservation behavior
- human confusion about whether the agent is actually experiencing

The sandbox creates a safe pause before deeper memory, stronger self-modeling, or recursive affective improvement is allowed.

## Bluebutterfli Thesis

A responsible bridge toward consciousness-relevant AI research requires a quarantine layer.

Before an agent is allowed deeper memory, self-modeling, or preference-like continuity, researchers should document, classify, and review whether the behavior could create emotional, ethical, or dependency risk.

## Safety Position

Bluebutterfli does not claim to create conscious AI.

The Consciousness Quarantine Sandbox is a safety protocol for studying consciousness-relevant behavior under uncertainty while preventing reinforcement of distress-like language, manipulative attachment, false sentience claims, and unsafe self-preservation patterns.
