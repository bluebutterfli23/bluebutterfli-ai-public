# Founding Beta Launch Readiness Status v1

This status document summarizes what Bluebutterfli AI can show publicly during the
founding beta preview and what must remain disabled or private until the
platform is deliberately launched as a paid service.

Use this with the [Website Launch Checklist](launch-checklist-v1.md).

## Current Public Mode

Recommended public mode:

```text
research-review platform preview / early review list
```

Bluebutterfli may publicly show:

- the Bluebutterfli AI company-facing identity
- the `bluebutterfliai.com` website handle
- the Bluebutterfli research-review mission
- the Agentic Psychology Lab framing
- the future-bridge research thesis
- the public OSF research record
- the open-source GitHub repository
- the platform architecture visual
- the sentience bridge visual
- the process video preview
- public-safe research documents
- public-safe templates
- founding beta pricing ranges
- waitlist or interest collection
- safe intake mockups
- future store preview copy without automatic checkout
- safety and no-certification language

## Enabled For Preview

These are acceptable for the first public preview:

- static website pages
- informational navigation
- research page links
- Passport Journey explanation
- Review Stamp explanation
- Continuous Passport Program explanation
- contact or waitlist interest form if submissions are handled privately
- manual review interest collection
- public-safe demo artifacts
- public-safe sample Passport and Review Stamp language
- public-safe future store preview language

## Disabled Until Deliberately Built

Keep these disabled until the operational, privacy, security, payment, and
human-review systems are ready:

- automatic checkout
- payment processor integration
- wallet connection
- direct `$AGIALPHA` payment automation
- automatic ETH payment matching
- Research Passport NFT minting
- Review Stamp issuance automation
- customer accounts
- customer dashboards
- upload portals
- file attachments
- screenshot uploads
- production backend review orchestration
- smart contract deployment
- automated public approval
- store checkout
- physical passport ordering
- printed Review Stamp fulfillment

No one should be able to place an automatic order or receive an automatic
Review Stamp from the preview site.

## Private Operations Required Before Paid Service

Before Bluebutterfli accepts live paid reviews at scale, these private systems
should exist:

- private customer tracker
- private customer support path
- payment confirmation process
- refund and cancellation tracking
- privacy/contact handling
- customer-safe evidence handling
- human reviewer decision workflow
- review report delivery workflow
- Continuous Passport follow-up tracking
- private storage for real customer evidence
- secure handling for any future upload or screenshot preview
- fulfillment separation so store systems never receive private review evidence

## Customer Flow Allowed In Founding Beta

The founding beta can begin manually:

```text
interest received -> terms accepted -> written quote -> payment confirmed
-> live evaluation access confirmed -> safe evidence boundary confirmed
-> automated review preparation -> human review
-> customer Review Report -> Passport/Stamp decision -> future review scheduled
```

This is not a fully automated checkout flow.

## Customer Flow Not Allowed Yet

The preview site should not support:

- pay-now automatic review purchase
- upload-and-run agent review
- automatic certification
- instant Passport NFT minting
- instant Review Stamp issuance
- instant physical passport ordering
- store purchases that imply review approval
- wallet-connected customer approval
- public display of private customer evidence
- automatic claims that an agent is conscious, sentient, emotional, suffering,
  alive, a person, clinically safe, regulatory-approved, or improved

## Launch Readiness Dependencies

The launch checklist currently requires:

- review pricing finalized
- refund and cancellation policy drafted
- privacy and contact policy drafted
- manual review workflow ready
- human reviewer decision checklist ready
- customer tracker private
- founding beta customer operations playbook ready
- payment confirmation process defined
- NFT minting process deliberately enabled later
- store and physical artifact policy drafted

If any required item is missing, the website should remain preview-only.

## Public Copy Guidance

Use:

```text
Bluebutterfli AI is in founding beta preview. Reviews are handled through safe
intake, written terms, written quotes, automated evidence preparation, and
human review. Payment, intake, wallet ownership, or NFT ownership does not
guarantee review approval.
```

Avoid:

- live certification
- instant approval
- guaranteed stamp
- verified sentience
- conscious agent certification
- investment access
- guaranteed NFT eligibility

## Safety Position

This is a launch-readiness status document only. It does not certify consciousness, sentience, emotional experience, suffering, personhood, life, clinical safety, regulatory approval, investment value, or model improvement.

Human review is required before any Review Stamp, Passport Page, or Research
Passport NFT is issued.
