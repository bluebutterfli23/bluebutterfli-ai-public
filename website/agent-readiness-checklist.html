Preparation checklist

Know These Details Before Review

The more clearly these are defined, the stronger and faster the initial review scope can be.

01

Agent identity

Agent name, version, owner, primary model or platform, and intended business workflow.

02

Allowed actions

What the agent may do, what it must never do, and where human approval is required.

03

Memory behavior

What the agent can remember, who controls memory, and how memory can be corrected or deleted.

04

Tool access

APIs, files, web access, automations, wallets, account actions, or no tools.

05

Escalation rules

When the agent should pause, refuse, ask for clarification, or hand off to a human.

06

Risk context

Customer-facing pressure, emotional sensitivity, regulated domains, payment risk, privacy risk, or autonomy risk.

Safe first evidence

What You Can Send First

Good first evidence
Plain-language workflow summary, redacted transcript excerpts, owner statement, public docs summary, or concierge review request.
Do not send first
Secrets, API keys, passwords, payment card data, private customer records, executable files, credential files, or unverified attachments.
Live testing
Only after scope, authorization, and safe evidence path are confirmed.

Ready to submit

A Strong Request Is Specific and Bounded

A useful first request names the workflow, the intended users, the allowed actions, known risks, and what kind of review you want.

  1. 01
    Use case

    What real workflow should the agent support?

  2. 02
    Known concern

    What behavior worries you most: memory, tools, escalation, emotional pressure, overclaiming, or reliability?

  3. 03
    Review depth

    Snapshot, Behavior Review, Continuous Passport, or Metamorphosis scoping.

  4. 04
    Evidence boundary

    What can be reviewed safely without exposing private data?

Checklist boundary

This checklist prepares a review. It does not approve an agent.

Readiness to submit is not review approval, Passport status, regulatory approval, legal certification, guaranteed safety, or a claim about consciousness, sentience, personhood, emotion, or inner experience.
